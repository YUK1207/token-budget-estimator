01 / Forecast cockpit

One sticky screen. Four product moments.

Like a device launch page, the product UI stays on stage while the story moves around it. Each scroll step changes the forecast screen and reveals a new part of the preflight decision.

Token Forecast

Predict the whole run, not just the prompt.

It accounts for file reads, searches, command output, failed tests, retries, analysis, and the final answer.

ROI Assessor

Decide whether the task deserves execution.

High-uncertainty work gets pushed toward discovery, decomposition, or a smaller first pass.

Budget Contract

Give Codex a boundary before it moves.

Readable files, command count, log limits, retry rules, and stop conditions become part of the execution plan.

Context Diet

Spend the first tokens where signal is highest.

The agent reads the nearest evidence first and expands only when the estimate says it is worth it.

02 / Product engine

Not a token counter. A decision system.

The useful part is not pretending to be exact. The useful part is turning uncertainty into a safer execution path.

Forecast Range

Outputs intervals, not fake precision.

Every estimate is a range because agent work is path-dependent. The system admits uncertainty instead of hiding it.

Risk Routing

Moves big tasks into smaller launches.

High-budget work gets pushed toward discovery, split plans, or bounded execution before the agent touches the repo broadly.

History Calibration

Learns from actual usage over time.

Recorded outcomes can tune future forecasts by task type, making the estimator more useful inside real workflows.
